What does it mean to be a good team member in software development? 

 

In my opinion, I believe that for someone to be a good team member in software development is to be as helpful to the team as you are in your task. Even if you successfully complete your tasks to the best of your abilities, if some of your teammates are struggling then the overall project might not be as good. Open communication and teamwork are critical to creating functional software. In the few software programs that I have developed so far, I have typically created them incrementally without an end goal, so as a result I would just add things that I think would be good as I think of them. I hope to incorporate more Agile processes into my methods when developing future projects. I hope to work with user stories and to set priorities when working on future final coding projects or any project I would like to brainstorm feature ideas before I start coding to have a clear direction to start implementing. I incorporate user needs into my programming because I am a user of some pieces of software and when I see choices that were made in developing software, I use professionally sometimes, I wonder why they chose to build it a certain way. Whether it be ease of use or the way the functionals are formatted in a way that is not user friendly. I think that using user stories would help developers see what is the basic functionality that they need to create and will provide some details on how it actually feels to operate.
